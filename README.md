- 160.相交链表：会做但不是最优解，比较优的解法是双指针。
- 236.二叉树的最近公共祖先：这个自己的思路是对的，但是可以改造原本的函数

## day 4

- 198.打家劫舍：这道题就是一个简单的动态规划，只需要记住一个点，找出当前范围内的最大可抢劫数，然后逐渐往后面累积即可。
- 169.多数元素：这道题似乎有一个众数的获取算法，我这里使用的是第k大元素这个思路，最优的情况下，时间复杂度是O(N)，似乎也可以，但是最坏的情况下需要O(N^2)的复杂度吧？似乎有点不太对，这里应该学习一下众数选举的算法。

## day 5

- 139.单词拆分：这个题目主要是需要熟练手写前缀树，然后要用一点动态规划的思路来做这道题。
- 141.环形链表：这个题目主要是需要记住数学推导，双指针很简单。
- 142.环形链表II：这个题目也是需要记住数学推导。
- 146.LRU缓存：这个题目主要是要熟练的写双链表，单链表为了方便操作第一个节点，往往会加上一个冗余的头节点，而双链表则可以再加上一个冗余的尾节点用来方便操作最后一个节点。然后就是要熟练封装，这样会比较容易来实现双链表的各项功能。
